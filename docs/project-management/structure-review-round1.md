# 1. 書籍の概要

- 想定読者（`docs/index.md` の記述ベース）
  - 計算可能性理論・計算量・物理的制約といった概念を前提に、「人間とAIの知能の等価性」を理論面から検討したい読者。
  - 哲学的論考の形式も含むため、入門者よりは「理論を読む体力」がある層に向く（初心者向けにする場合は前提の再整理が必要）。
- 本書で達成しようとしているゴール（`docs/index.md` の記述ベース）
  - 計算論的物理主義の主張・前提・用語を説明できる。
  - 計算可能性/計算量/物理法則の制約の関係を整理し、原理的な計算可能範囲を議論できる。
  - 既存のAI観（強いAI/弱いAIなど）との比較の中で本書の立場を位置づける。
  - 実装可能性や倫理・社会的含意まで含めて見解を構築する。

# 3. 構成上の気になる点（候補）

- 章頭の学習目標が明示されていない
  - 章の導入はあるが、「この章でできるようになること」が章頭に固定化されていないため、読者が自己評価しにくい。
- 見出しレベルの一貫性（`#` の混在）
  - 章内に `# ...` の見出しが登場し、章タイトルと同一レベルになっている箇所がある（例: 第1章の PAC 学習例、第3章の比較図示など）。目次や読書の区切りとして誤認される可能性がある。
- 理論→実装→含意の接続導線
  - 章順は自然だが、第2章（厳密化）→第4章（実装）への接続で「どの定理/定義がどの実装要素に対応するか」を章頭/章末で再掲すると、追従しやすい。
- ライセンス表記の不整合
  - `docs/index.md` は “All rights reserved.” だが、`LICENSE.md` は CC BY-NC-SA 4.0（シリーズ統一ライセンス準拠）。読者の信頼性に関わるため整合が必要。

# 4. 構成改善の提案（案）

- 章頭に「学習目標（できるようになること）」を追加する
  - 各章 3〜5 点で、学習目標を固定化する（現状の導入段落を要約するだけでも効果が高い）。
- 見出しレベルの正規化（ToCの安定化）
  - 章内の補助見出しは `##` / `###` に揃え、`#` は章タイトルに限定する。
- 章間の対応関係を可視化する
  - 第2章末に「定義/定理→第4章の実装要素」対応表を置き、第4章冒頭で参照する（読者の迷子防止）。
- ライセンス表記を `LICENSE.md` と整合させる（低リスク）
  - `docs/index.md` のライセンス節をシリーズ統一表記へ揃える。
