# 実験の実行手順（最小）

本ディレクトリは、本書（第4章）の「最小実装＋評価ハーネス」を提供します。  
ここでの目的は、実務で頻出する **生成→評価→選択** のループを、手元で「動く」形で確認することです。

## 前提

- Node.js 20+

## 1. 推論時（test-time）計算量スケーリングの最小例

三目並べ（Tic-Tac-Toe）で、MCTS（Monte Carlo Tree Search）の**シミュレーション回数**を増やすと、勝率が上がるかを確認します。  
ここでは、1手あたりのシミュレーション回数を **推論時計算量のproxy** として扱います。

```bash
node experiments/benchmarks/tic-tac-toe/run-mcts-vs-random.js --games 200 --sims 10,50,200 --seed 1
```

出力は、`--sims` の各値に対して、勝敗（win/draw/loss）と所要時間を表示します。

## 注意

- これは**おもちゃ問題**です。実際のLLMでは「トークン数」「探索分岐」「外部ツール呼び出し」「wall-clock」「コスト」など複数のproxyを併用してください。
- 乱数シードを固定しても、実行環境差（CPU性能等）で時間は変わります。
