# Metrics（計測の最小方針）

本書では「知性＝計算量」の議論を、実務に落とすために次の2軸で測ることを推奨します。

## 1) 品質（quality）

例：

- 正解率 / 再現率 / BLEU 等（タスク依存）
- 期待報酬（強化学習的）
- 勝率（ゲーム）

## 2) 計算資源（compute; proxy）

例：

- 推論トークン数（入力/出力/再試行/自己検証を含む）
- 探索分岐数（サンプリング回数、ロールアウト回数、ビーム幅）
- 外部ツール呼び出し回数（検索/RAG、コード実行、DB問い合わせ等）
- wall-clock（レイテンシ、スループット）
- 推定電力/コスト（GPU時間、消費電力推定、クラウド課金額）

## 重要な注意

- 1つのproxyだけで「知性」を定義しないでください。品質と資源のトレードオフを測ることが目的です。
- 再現性のために、入力・乱数・環境情報（CPU/GPU、ライブラリ版）をログに残してください。
